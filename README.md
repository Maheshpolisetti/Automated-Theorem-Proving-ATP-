# Automated Theorem Prover for Set Theory

## Project Overview
This project addresses the challenge of automated reasoning and formal mathematical verification by developing a logic agent capable of Automated Theorem Proving (ATP) within an axiomatically constrained subdomain of Set Theory. 

Automating mathematical proofs is a prominent problem in Artificial Intelligence. It eliminates human error in complex logical derivations, accelerates mathematical research by verifying intricate proofs, and acts as a pivotal stepping stone toward more generalized automated reasoning systems for wider domains of abstract mathematics (like number theory, topology, and algebra). The significance of this interdisciplinary project spans potential advancements not only in pure mathematics but also across various computer science domains that inherently rely on discrete structures.

## Operational Pipeline
The system strictly operates utilizing a **Knowledge Base (KB)** consisting of a constrained version of set theory axioms. The agent accepts a target theorem or hypothesis to be evaluated and provides a definitive boolean conclusion determining whether the theorem is a logical consequence of the axioms, followed by a clear, step-by-step trace of the proof if true.

To achieve this, the system structurally manipulates the state-space:
1. **Translation:** The Knowledge Base and the structurally negated hypothesis are parsed into Conjunctive Normal Form (CNF) via AST mappings, dropping implicit universals smoothly into a matrix format.
2. **State-Space Search:** The derivation unfolds as a state-space search where the initial state contains the generated CNF clauses, and the terminal goal state is the **Empty Clause (`□`)**, mathematically certifying a logical contradiction against the negated goal.
3. **Execution:** The agent navigates this search space strictly using binary resolution, structural factoring, and a native symmetric equality mapping algorithm (Paramodulation).


## Algorithmic Foundation
The core theoretical engine behind the solver relies strictly on **First-Order Logic Resolution Refutation**. Theorem proving inherently requires verifying if a conclusion functionally follows from a set of axioms. Resolution refutation accommodates this structure flawlessly by attempting to prove that the negation of the hypothesis is structurally inconsistent with the Knowledge Base.

### Managing Combinatorial Complexity (Optimizations)
A notorious challenge in ATP systems is the exponential combinatorial complexity of the resolution branch tree. The engine integrates multiple sophisticated heuristics and pruning methodologies to survive the expansion logic:

- **Set of Support (SoS) Strategy:** Drastically restricts the search space by algorithmically enforcing that at least one of the parent clauses being resolved dynamically maps back to the negated target goal. This strictly prevents the system from aimlessly bouncing background truths off each other forever.
- **Shortest Clause Preference:** Actively penalizes bloated clauses, prioritizing shorter derivations internally via rigorous queue sorting to push toward the isolating empty clause faster.
- **Tautology Deletion:** Aggressively prunes the active search tree by identifying and isolating logically redundant (tautological) clauses immediately.

### Transparency Advantage
The hallmark advantage of a First-Order Logic (FOL) resolution engine is that it strictly defies the "black box" nature of typical AI implementations. Because the engine runs natively on strictly defined, rule-based mechanics, every single mathematical proof generated by the program showcases explicit logical steps mapping back flawlessly to the foundational parent assumptions.

## Base Knowledge Matrix (7 Axioms)
The system currently limits its functional domain native baseline to the following 7 axioms:

1. **Empty Set**: Nothing is in the EmptySet
2. **Universal Set**: Everything is in the UniversalSet
3. **Intersection**: `x in (A ∩ B)` iff `(x in A AND x in B)`
4. **Union**: `x in (A ∪ B)` iff `(x in A OR x in B)`
5. **Subset**: `A is subset of B` iff `(for all x, x in A implies x in B)`
6. **Difference**: `x in (A \ B)` iff `(x in A AND x not in B)`
7. **Equality (Extensionality)**: `A equals B` iff `(A subset B AND B subset A)`
